## What v44 did NOT fix тАФ new failure modes found

### 1. Gemini Flash internal `_recoverFromLoop` self-abort
**File:** `outputs/dashboard-runs/ba6c256185e3/readme-deploy.err.md`

```
AbortError: The user aborted a request.
    at GeminiClient._recoverFromLoop
    (тАж @google/gemini-cli/bundle/chunk-UN6XCVMJ.js:303894 тАж)
```

Gemini Flash detected itself entering a generation loop (probably
re-emitting the same Markdown header repeatedly while streaming the
README + DEPLOY) and **its own internal safety mechanism aborted the
turn**. This is not our timeout тАФ it's Gemini-side.

The output that DID land before abort (6.3 KB) is real and useful тАФ a
solid README draft for the `cctok` project including title, hero
placeholder, feature list, and the start of a Fly.io deploy guide.

**Mitigation idea (next iteration):** swap readme-deploy from Gemini
Flash to Claude Sonnet 4.6. Same speed tier for short Markdown,
no loop detector, more predictable.

### 2. Claude Sonnet 4.6 tests step hung at 0 B for 6 min
**File:** `outputs/dashboard-runs/ba6c256185e3/tests.out.md` (0 bytes)
**Stderr:** none (no `tests.err.md` created)

Sonnet subprocess started, never wrote a single character to stdout,
never errored. Most likely Anthropic API queueing under load (Pro+
subscription rate limiting) or a stream-json buffering glitch.

**Mitigations to consider:**
- Lower the agent timeout for tests step specifically (e.g. 240 s) so
  CG fails fast instead of waiting the full 720 s default
- Add a "first-token watchdog": if no output in 90 s, kill + mark
  failed
- Simpler fallback: shrink the test-step prompt (currently it includes
  the full 117 KB implementation as `{{implementation}}`, which may
  bump up against an API soft-limit)

### 3. Long-context inputs slow Sonnet drastically
The `tests` step receives `${director} (12.5 KB) + {{implementation}}
(117 KB)` = ~130 KB prompt. Even if it doesn't hang, this is right at
the edge of Sonnet's reliable handling.

**Mitigation:** rewrite the tests prompt to take only the director's
spec + a synopsis of implementation (e.g. file tree + signatures),
not the full 117 KB code dump.
