# deploy/

Versioned templates + scripts for the claudegravity.space production
setup. Full architecture + runbook: `docs/DEPLOY-LIVE.md`.

```
deploy/
├── vps/
│   └── Caddyfile.snippet      Append to /etc/caddy/Caddyfile on the VPS,
│                              then `systemctl reload caddy`.
└── windows/
    ├── install-tunnel.ps1     One-shot Scheduled Task installer for the
    │                          SSH reverse tunnel (preferred path).
    ├── cg-tunnel.ps1          Foreground tunnel loop (alt to install-tunnel,
    │                          handy for debugging — runs ssh in a while loop
    │                          with logging).
    └── frpc.toml.example      frp client config kept for reference. Not in
                               use because the VPS provider firewalls inbound
                               port 7000; SSH reverse tunnel won.
```

These are templates + scripts only. They contain no secrets. The
real auth credentials live in:

- The user's Desktop file `claudegravity-login.txt` (one copy)
- Two user-scope environment variables on the dashboard host
  (`CG_AUTH_USER` + `CG_AUTH_PASSWORD_HASH`)

Both are generated by `python src/cg.py auth init`.
